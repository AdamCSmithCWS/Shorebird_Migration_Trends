Thank you for downloading data through Naturecounts.

The file you have downloaded (titled BMDE_data.txt) is a text file in CSV format. This format minimizes the size of your download and allows users to import the data into almost all data management programs.

To Open the file in Microsoft Excel please follow the steps below:

  • save the file on your computer
  • open Microsoft Excel
  • choose 'File'
  • 'Open' from the menu
  • navigate to the folder in which the BMDE_data.txt file is saved
  • change the 'Files of Type' tab at the bottom of the window to display 'All files'
  • chose the BMDE_data.txt file and click 'Open'
  • a window will open that will allow you to convert the CSV file to excel format
  • choose 'delimited' and click next
  • select 'Tab' and click Next
  • click Finish

You should now have the data in excel format. If you need further instructions or assistance please contact Catherine Jardine at cjardine@birdscanada.org

Please note that Excel 2003 or earlier versions can only open up to 65,000 records. Starting with Excel 2007, the limit exceeds 1 million rows. If the file exceeds the Excel limitations, you may have to open the text file in a text editor and create several smaller files, but a better alternative would be to use a program that can handle larger data files, such as Microsoft Access.

Notes on data format.

The data structure of naturecounts data is Bird Monitoring Data Exchange, the standard format for the Avian Knowledge Network.

Information on the data fields contained in this format can be found here: http://www.birdscanada.org/resources/default/bmde2.00.pdf

Details on the collection methods and scope of each of dataset can be found by browsing the list of datasets available through Naturecounts here: http://www.birdscanada.org/birdmon/default/datasets.jsp

After selecting a project you can browse the associated metadata or link to the project website where additional details are often available.

Please ensure that any use of the data is appropriately acknowledged. Refer to the datapolicy file provided with each dataset for details.
